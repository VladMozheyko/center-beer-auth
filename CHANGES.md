## Эндпоинты с изменениями

<a name="вход-через-социальную-сеть"></a>
### <span style="color: #28a745">Вход через социальную сеть</span>

**Эндпоинт:** `POST /oauth2/social/login`

**Тип:** 🚨 modify (изменен)

**Описание:** Вход через социальную сеть

**Изменения:**
- Добавлено новое поле в запросе `deviceId` - необязательное (identifier устройства)
- Добавлено новое поле в ответе `lastIpAddress` - содержит IP-адреса за последние 5 минут, с которых пользователь совершал вход

**Пример ответа:**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "d0f8d9c3-3b4a-4c89-9a3a-1a2b3c4d5e6f",
  "deviceId": "device-8f7a1b2c-3d4e-5f6a-7b8c-9d0e1f2a3b4c",
  "message": "Успешный вход через соцсеть GOOGLE",
  "email": "user@example.com",
  "status": "SUCCESS",
  "lastIpAddress": [
    {
      "createdAt": "2026-07-01T07:47:05.193Z",
      "ipAddress": "200.10.80.33"
    },
    {
      "createdAt": "2026-07-01T07:45:00.000Z",
      "ipAddress": "192.168.1.100"
    }
  ]
}
```

**Запрос:**
```json
{
  "authCode": "abcdefgh",
  "provider": "GOOGLE",
  "deviceId": "ABCDEF12-3456-7890-abcd-ef1234567890"
}
```

<a name="привязка-социальной-сети"></a>
### <span style="color: #28a745">Привязка социальной сети</span>

**Эндпоинт:** `POST /oauth2/social/link`

**Тип:** 🚨 modify (изменен)

**Описание:** Привязка социальной сети к аккаунту (требует авторизации)

**Изменения:**
- Добавлено новое поле в запросе `deviceId` - необязательное (identifier устройства)
- Добавлено новое поле в ответе `lastIpAddress` - содержит IP-адреса за последние 5 минут, с которых пользователь совершал вход

**Пример ответа:**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "d0f8d9c3-3b4a-4c89-9a3a-1a2b3c4d5e6f",
  "deviceId": "device-8f7a1b2c-3d4e-5f6a-7b8c-9d0e1f2a3b4c",
  "message": "Соцсеть VK привязана",
  "email": "user@example.com",
  "status": "SUCCESS",
  "lastIpAddress": [
    {
      "createdAt": "2026-07-01T07:47:05.193Z",
      "ipAddress": "200.10.80.33"
    }
  ]
}
```

**Запрос:**
```json
{
  "authCode": "abcdefgh",
  "provider": "GOOGLE",
  "deviceId": "ABCDEF12-3456-7890-abcd-ef1234567890"
}
```

**Требования:**
- Пользователь должен быть авторизован (JWT в куки)
- Привязывает соцсеть к текущему аккаунту

<a name="регистрация-через-социальную-сеть"></a>
### <span style="color: red">Регистрация через социальную сеть</span>

**Эндпоинт:** `POST /oauth2/social/register`

**Тип:** new (новый эндпоинт)

**Описание:** Регистрация через социальную сеть

**Запрос:**
```json
{
  "authCode": "abcdefgh",
  "provider": "VK",
  "deviceId": "ABCDEF12-3456-7890-abcd-ef1234567890"
}
```

**Поля запроса:**

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| `authCode` | `string` | Да | Одноразовый код из редиректа |
| `provider` | `string` (`GOOGLE`, `YANDEX`, `VK`) | Да | Провайдер авторизации |
| `deviceId` | `string` | Нет | ID устройства |

**Ответ:**
Тот же, что и `login` - с `lastIpAddress`

## Новые эндпоинты

---

### <span style="color: red">Получение IP-адресов регистрации пользователя</span>

**Эндпоинт:** `GET /user/{id}/registration/ips`

**Тип:** new (новый эндпоинт)

**Описание:** Получение IP-адресов, с которых пользователь совершал вход за последние 5 минут

**Параметры:**

| Параметр | Тип | Обязательное | Описание |
|---------|-----|-------------|----------|
| `id` | `long` | Да | ID пользователя |

**Ответ:**
```json
[
  {
    "createdAt": "2026-07-01T07:47:05.193Z",
    "ipAddress": "200.10.80.33"
  },
  {
    "createdAt": "2026-07-01T07:45:00.000Z",
    "ipAddress": "192.168.1.100"
  }
]
```

**Описание полей:**

| Поле | Тип | Описание |
|------|-----|----------|
| `createdAt` | `datetime` | Время регистрации IP-адреса |
| `ipAddress` | `string` | IP-адрес |

**Пример запроса:**
```
GET /user/123/registration/ips
```

<a name="выход-из-системы-с-текущего-устройства"></a>
### <span style="color: red">Выход из системы с текущего устройства</span>

**Эндпоинт:** `POST /auth-session/logout`

**Тип:** new (новый эндпоинт - замена устаревшему)

**Описание:** Выход из системы только с текущего устройства

**Параметры:** Нет

**Ответ:**
- `200 OK` - успешный выход

**Особенности:**
- В отличие от `/authentication/logout`, этот эндпоинт четко указывает, что выход производится только с текущего устройства
- Поддерживает до 3 одновременных входов на разных устройствах

---

### <span style="color: red">Выход из системы со всех устройств</span>

**Эндпоинт:** `POST /auth-session/logout/all`

**Тип:** new (новый эндпоинт)

**Описание:** Выход из системы на всех устройствах (отзыв всех refresh-токенов)

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|---------|-----|-------------|----------|
| `loggingOutOfThisDevice` | `boolean` | `false` | Флаг, указывающий, выходит ли пользователь и с этого устройства |

**Пример запроса:**
```
POST /auth-session/logout/all?loggingOutOfThisDevice=true
POST /auth-session/logout/all?loggingOutOfThisDevice=false
```

**Описание параметров:**
- `loggingOutOfThisDevice=false` - выход со всех устройств, кроме текущего
- `loggingOutOfThisDevice=true` - выход со всех устройств, включая текущее

**Ответ:**
- `200 OK` - успешный выход со всех устройств

### <span style="color: red">Список активных сессий</span>

**Эндпоинт:** `GET /auth-session/sessions`

**Тип:** new (новый эндпоинт)

**Описание:** Получение списка активных сессий/устройств пользователя

**Ответ:**
```json
[
  {
    "id": 1,
    "token": "d0f8d9c3-3b4a-4c89-9a3a-1a2b3c4d5e6f",
    "expiryDate": "2026-07-15T07:47:05.193Z",
    "revoked": false,
    "createdAt": "2026-07-01T07:47:05.193Z",
    "lastUsedAt": "2026-07-01T07:47:05.193Z",
    "deviceInfo": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
  }
]
```

**Описание полей:**

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `long` | ID refresh-токена |
| `token` | `string` | Значение refresh-токена |
| `expiryDate` | `datetime` | Дата истечения срока действия |
| `revoked` | `boolean` | Флаг отозванности токена |
| `createdAt` | `datetime` | Дата создания |
| `lastUsedAt` | `datetime` | Дата последнего использования |
| `deviceInfo` | `string` | Информация об устройстве (User-Agent) |

### <span style="color: red">Профиль пользователя</span>

**Эндпоинт:** `GET /user/profile/{id}`

**Тип:** new (новый эндпоинт)

**Описание:** Получение полной информации о пользователе (для геймификации)

**Параметры:**

| Параметр | Тип | Обязательное | Описание |
|---------|-----|-------------|----------|
| `id` | `long` | Да | ID пользователя |

**Ответ:**
```json
{
  "userName": "string",
  "email": "string",
  "phone": "string",
  "socialAccounts": [
    {
      "provider": "VK",
      "socialEmail": "string"
    }
  ],
  "fileData": {
    "name": "string",
    "path": "string"
  }
}
```

**Описание полей:**

| Поле             | Тип                           | Описание                      |
|------------------|-------------------------------|-------------------------------|
| `userName`       | `string`                      | Никнейм пользователя          |
| `email`          | `string`                      | Email пользователя            |
| `phone`          | `string`                      | Телефон пользователя          |
| `socialAccounts` | `List<SocialAccountResponse>` | Данные о социальных аккаунтах |
| `fileData`       | `FileDataDto`                 | Данные о картинке             |

## Устаревшие эндпоинты

---

### <span style="color: yellow">Устаревший выход из системы</span>

**Эндпоинт:** `POST /authentication/logout`

**Тип:** deprecated (устарел)

**Описание:** Выход из системы

**Причина устаревания:**
- Неявное поведение - не указано, с каких устройств происходит выход
- Заменен на более явный эндпоинт `/auth-session/logout` с четким описанием

**Рекомендуемая замена:**
- Использовать `/auth-session/logout` для выхода с текущего устройства
- Использовать `/auth-session/logout/all` для выхода со всех устройств

## Технические детали

### DTO Changes

**AuthenticationResponseDto** - добавлено поле:
```java
private List<UserIpTempDto> lastIpAddress;
```

**SocialExchangeRequest** - добавлено поле:
```java
private String deviceId;
```

**UserIpTempDto** - используется для хранения информации об IP-адресах:
```java
private Instant createdAt;
private String ipAddress;
```

- **0.0.0** - предыдущая версия с базовой аутентификацией
